论程序员的自我修养

2019-06-06
  • 软件随笔
  • 程序员 自我修养

# 学习

不是为了求得答案，而是了解原理和获得方法。 持续学习、终身学习是知识爆炸年代的不二选择。 学习不是人云亦云，而是总结、思辨、践行的过程。

# 读优秀源代码胜于各种理论

和工作结合，如java领域的同学可以从spring framework，spring boot入手，一次读一块，后续可以读sping cloud各个项目源代码，读知名中间件源代码，读工作中遇到实际问题的源代码，关键是要形成文字总结、分享。 比如，你在微服务调用过程中遇到了文件传输的需求和问题，为了彻底搞懂理论和细节，你可以专题阅读HttpMessageConverter相关代码

# 有意识学习技术技巧，写demo代码

如：

  • 利用python的pyitotify实现文件监控
  • python3中singleton实现
  • java8的lambda使用
  • javascript的promise实现
  • nginx启用lua
  • redis内存组织
  • mq的消息存储
  • mysql的索引

有条件，最好固定时间，如每周、每月刷算法题，推荐其中一个网站：leetcode。leetcode支持选择不同开发语言在线答题：

算法题最考验编程基本功，让你的技法不至于生疏。也让你了解其它知名软件的内部实现机制，比如你如果自己写过红黑树算法，你会对java8中ConcurrentHashMap实现机制就会更了解。

# 分享

把心得写下来，有机会描述给别人听，这对你的提高不是一点半点！积累到一定程度，可以在公司或其它层面进行授课分享。分享是最有效的知识梳理沉淀过程。

# 应用

学以致用。谨慎评估，大胆使用。在应用技术的过程中会实现公司的价值增长，也会实现你的个人价值。

# 坚持

李笑来曾经总结：学习上的成功，都只靠两件事：策略和坚持，而坚持本身就是最重要的策略。 坚持必有所获。
